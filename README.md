# Company Portal – Внутренний портал компании
Видео-демо: https://drive.google.com/drive/folders/1bbCuxZ4mv6vL72vRi8GmMgDmoL8szSSo?usp=sharing

Учебный проект веб-приложения для корпоративного взаимодействия.

## Технологии
- **Фронтенд:** HTML5, CSS3 (SCSS), JavaScript (Fetch API)
- **Бэкенд:** PHP 8.x, MySQL
- **Сервер:** Apache 2.4 (Windows)
- **Безопасность:** хеширование паролей (password_hash), подготовленные SQL-запросы, сессии
- **Дополнительно:** звуковые эффекты (HTML5 Audio), анимации CSS, адаптивный интерфейс

## Установка и запуск (локально)

### Предварительные требования
- Установленный веб-сервер Apache 2.4
- PHP версии 8.x с расширением `mysqli`
- Git (для клонирования репозитория)

### 1. Клонирование репозитория
Откройте командную строку (Git Bash, PowerShell или cmd) и выполните:
```
git clone https://github.com/verivanova/company-portal.git
cd company-portal
```

### 2. Настройка Apache
Откройте конфигурационный файл Apache (httpd.conf).
Укажите путь к папке проекта в директиве DocumentRoot.
```
DocumentRoot "C:/путь/к/company-portal"
<Directory "C:/путь/к/company-portal">
    Options Indexes FollowSymLinks
    AllowOverride None
    Require all granted
</Directory>
 ```

Подключите PHP как модуль:
```
LoadModule php_module "C:/php/php8apache2_4.dll"
PHPIniDir "C:/php"
AddType application/x-httpd-php .php
```

Директива ServerName должна быть установлена как localhost:80

Перезапустите Apache: `C:\Apache24\bin\httpd.exe -k restart` (через командную строку от имени администратора)

### 3. Настройка базы данных
1) Создайте пустую базу данных MySQL. Например, через консоль MySQL или MySQL Workbench выполните:
```
CREATE DATABASE company_portal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
3) Создайте все необходимые таблицы. Для этого в папке backend/ есть скрипты, которые нужно выполнить однократно, открывая их в браузере:

- http://localhost/backend/setup_tasks.php – создаст таблицу tasks

- http://localhost/backend/setup_news.php – таблицу news

- http://localhost/backend/setup_action_logs.php – таблицу action_logs

- http://localhost/backend/add_locked_field.php – добавит поле is_locked в таблицу users

! После выполнения каждого скрипта удалите его из папки backend/ в целях безопасности

### 4. Файл конфигурации
Создайте и откройте `backend/config.php` и введите свои реальные данные для подключения к MySQL:
```php
<?php
$host = 'localhost';
$user = 'root';         // ваш MySQL
$password = 'пароль';   // пароль MySQL
$database = 'company_portal'; // имя созданной базы данных

$mysqli = new mysqli($host, $user, $password, $database);
if ($mysqli->connect_error) {
    die('Ошибка подключения: ' . $mysqli->connect_error);
}
$mysqli->set_charset('utf8mb4');
```
### 5. Добавление тестовых пользователей
Выполните скрипт `backend/seed_users.php` (откройте его в браузере один раз):
http://localhost/backend/seed_users.php

После этого также удалите файл.

### 6. Запуск
Перезапустите Apache:
```
C:\Apache24\bin\httpd.exe -k restart
```
Откройте в браузере http://localhost. Войдите под одной из тестовых учётных записей.

##  Основные функции
- Авторизация и сессии: вход с проверкой роли и блокировок. Защищённые страницы доступны только авторизованным пользователям.

- Лента новостей: публикация и удаление новостей. Удалять новость может автор или администратор.

- Управление задачами: создание, редактирование, удаление, смена статуса (выполнена/не выполнена).
-  Назначение постановщика и ответственного из реальных пользователей.
-  Валидация срока выполнения (не раньше сегодня и не позже +1 года). Просроченные задачи выделяются красным цветом и подписываются «Просрочена».

- Панель администратора: добавление новых пользователей с выбором роли, блокировка/разблокировка, удаление. Админ не может заблокировать или удалить сам себя.

- Журнал действий: логирование операций (добавление/удаление/изменение пользователей, задач, новостей, смена статусов) за последние 14 дней. Отображается имя выполнившего действие, дата и время.

- Звуковые уведомления: воспроизводятся при успешных действиях, удалении, ошибках. Реализован менеджер звуков с возможностью отключения и настройки громкости через localStorage.

- Тёмная/светлая тема: переключается кнопкой в шапке, выбор сохраняется между сессиями.

- Адаптивный интерфейс: корректное отображение на мобильных устройствах и планшетах.

- Обратная связь: анимации появления и удаления элементов (fadeIn/fadeOut).
